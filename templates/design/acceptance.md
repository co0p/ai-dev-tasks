# Validation Checklist (Design)

Before approving the design:

- Alignment: Traces to `increment.md` and `CONSTITUTION.md`.
- ADRs: Explicitly aligns with any existing `adr-*.md` decisions; call out references or conflicts and resolutions. This is mandatory.
- Boundaries: Clear module responsibilities and interfaces.
- Testability: Identified test points and success signals.
- Risks: Assumptions listed with mitigations.
- Feasibility: Implementable in small steps.
- Date: Use YYYY-MM-DD for date fields.

- Codebase Fit: Loosely align with existing code structure and patterns; note any intentional deviations.
